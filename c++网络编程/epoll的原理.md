epoll的原理

**epoll** 是 Linux 内核中的一种高性能 I/O 事件通知机制，旨在解决传统的 select 和 poll 在处理大量并发连接时的性能瓶颈。它的核心特点包括：

- **高效性**：epoll 通过使用红黑树和链表来管理文件描述符，能够有效地处理大量的文件描述符。

- 当文件描述符的内核缓冲区非空时，epoll 会发出可读信号进行通知；当写缓冲区不满时，发出可写信号通知。 

- API 接口：epoll 提供了三个主要的系统调用：epoll_create、epoll_ctl 和 epoll_wait，用于创建 epoll 实例、控制事件和等待事件发生。 

- 适用场景：epoll 特别适用于高并发的网络服务器和分布式系统，能够显著提高 CPU 利用率。
