左值 有明确存储位置 可以取地址

右值 比如字面常量10,3.14,”hello“、临时对象、表达式的计算结果等 没办法找到固定存储位置

C++ 标准规定：当右值引用绑定到一个临时对象时，该临时对象的生命周期会被延长至与右值引用相同，避免临时对象过早销毁。

std::move 将左值转成右值引用，可以使用移动语义，调用移动构造函数或移动赋值运算符。具体取决于目标对象是否实现移动赋值运算符、移动构造函数，没有实现的话，仍然是深拷贝

```c++
template <typename T>
// std::remove_reference<T>::type  移除T的引用修饰符
constexpr typename std::remove_reference<T>::type&& move(T&& t) noexcept {
    return static_cast<typename std::remove_reference<T>::type&&>(t);
}
```

std::forward 完美转发，在模板函数的参数转发上，能够将参数以其原始的左值或者右值属性传递给下一个函数，不改变参数的类型 



（通用引用T&&）

当形参是 **T&& 且 T 需要推导时（转发引用）**：

- 如果实参是左值表达式，T 被推导为 **U&**
- 如果实参是右值表达式，T 被推导为 **U**
   然后 `T&&` 再经过引用折叠得到最终参数类型。

  **引用折叠规则**：

有左就是左值引用，否则才是右值引用

- `U&  &&` 折叠成 `U&`

- U &&` 折叠成 `U&&

  实参进入转发引用 `T&&` 时，按特殊规则推导出 `T`；再通过引用折叠确定 `T&&` 的最终引用类型；最后 `std::forward<T>` 用 `static_cast<T&&>` 按 `T` 恢复原始值类别并转发给下游。

```c++
template<class T>
constexpr T&& forward(typename std::remove_reference<T>::type& t) noexcept {
    return static_cast<T&&>(t);
}

template<class T>
constexpr T&& forward(typename std::remove_reference<T>::type&& t) noexcept {
    static_assert(!std::is_lvalue_reference<T>::value, "bad forward call");
    return static_cast<T&&>(t);
}
```



```c++
void process(int& x)  { std::cout << "lvalue\n"; }
void process(int&& x) { std::cout << "rvalue\n"; }
template<typename T>
void wrapper(T&& arg) {
    // process(arg);               //  错误 arg左值 永远触发到左值处理lvalue
    process(std::forward<T>(arg)); // 保持arg的原始类别
   
}
```
