内存对齐
元素的起始地址放在类型大小的整数倍的位置上，不满足则需要填充字节
作用:
比如一个int 起始地址从1开始，访问的时候，就需要加载0-3 4-7两次，剔除不需要的数据，剩余数据合并出来，访问效率低，故而内存对齐可以提高性能，高效读写内存。

规则：
编译器进行对齐
*基本类型的对齐值就是其sizeof值
*结构体的对齐值是其成员中的最大对齐值，结构体总大小就是最大对齐值的整数倍
*编译器可以设置最大对齐值，此时实际对齐值 = min(默认对齐值，设置对齐值)
```c++
#pragma pack(n) 
// 设置最大对齐值
alignas(64) 
 // 强制把变量的起始地址对齐到64字节边界
```

现代主流cpu的cache line大小通常是64字节，SPCP中，alignas(64) 可以保证 生产者和消费者线程需要访问的两个下标位于不同的cache line，防止伪共享问题，即

线程之间没有共享变量，但是变量位于同一个cache line ，这样对变量的访问修改就需要在不同核心中做同步，反复标记为脏，降低性能。
